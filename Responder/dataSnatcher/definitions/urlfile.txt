<p></p><h3><b>compression definition</b></h3><br>1. <application> (Or "compaction") The coding of data to save storage space or transmission time. Although data is already coded in digital form for computer processing, it can often be coded more efficiently (using fewer bits). For example, <a id="dic_result" href="http://dictionary.reference.com/browse/run-length%20encoding">run-length encoding</a> replaces strings of repeated characters (or other units of data) with a single character and a count. There are many compression <a id="dic_result" href="http://dictionary.reference.com/browse/algorithms">algorithms</a> and utilities. Compressed data must be decompressed before it can be used. <br>The standard <a id="dic_result" href="http://dictionary.reference.com/browse/Unix">Unix</a> compression utilty is called <a id="dic_result" href="http://dictionary.reference.com/browse/compress">compress</a> though <a id="dic_result" href="http://dictionary.reference.com/browse/GNU">GNU</a>'s superior <a id="dic_result" href="http://dictionary.reference.com/browse/gzip">gzip</a> has largely replaced it. Other compression utilties include <a id="dic_result" href="http://dictionary.reference.com/browse/pack">pack</a>, <a id="dic_result" href="http://dictionary.reference.com/browse/zip">zip</a> and <a id="dic_result" href="http://dictionary.reference.com/browse/PKZIP">PKZIP</a>. <br>When compressing several similar files, it is usually better to join the files together into an <a id="dic_result" href="http://dictionary.reference.com/browse/archive">archive</a> of some kind (using <a id="dic_result" href="http://dictionary.reference.com/browse/tar">tar</a> for example) and then compress them, rather than to join together individually compressed files. This is because some common compression <a id="dic_result" href="http://dictionary.reference.com/browse/algorithms">algorithms</a> build up tables based on the data from their current input which they have already compressed. They then use this table to compress subsequent data more efficiently. <br>See also <a id="dic_result" href="http://dictionary.reference.com/browse/TIFF">TIFF</a>, <a id="dic_result" href="http://dictionary.reference.com/browse/JPEG">JPEG</a>, <a id="dic_result" href="http://dictionary.reference.com/browse/MPEG">MPEG</a>, Lempel-Ziv Welch, "<a id="dic_result" href="http://dictionary.reference.com/browse/lossy">lossy</a>", "<a id="dic_result" href="http://dictionary.reference.com/browse/lossless">lossless</a>". <br>Compression FAQ (ftp://rtfm.mit.edu/pub/usenet/news.answers/compression-faq/). <br>Web Content Compression FAQ (http://perl.apache.org/docs/tutorials/client/compression/compression.html). <br><a id="dic_result" href="http://dictionary.reference.com/browse/Usenet">Usenet</a> newsgroups: news:comp.compression, news:comp.compression.research. <br>2. <multimedia> Reducing the dynamic range of an audio signal, making quiet sounds louder and loud sounds quieter. Thus, when discussing digital audio, the preferred term for reducing the total amount of data is "compaction". Some advocate this term in all contexts. <br>(2004-04-26) <br></multimedia></application>
